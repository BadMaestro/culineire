Владелец (2026-07-13) ввёл **постоянное правило**: перед тем как писать что-то
новое — сначала **аудит приложения**, вдруг это уже реализовано ранее.

**Why:** дважды всплывало, что «новая» фича уже частично сделана (центральный
спонсор через `get_sponsor_of_month()` уже штамповал битвы; сезонный движок).
Дублирование = лишний код и конфликты.

**How to apply:**
- Перед новыми моделями/сервисами: `grep` по всему коду на близкие сущности,
  проверить существующие helpers/context-processors/signals.
- Не путать похожие, но разные сущности: напр. `Faction(kind=specialty)`
  (идентичность шефа) ≠ `Recipe.Category.GRILLING_AND_BARBECUE` (тег контента) —
  разные модели, разные namespace.
