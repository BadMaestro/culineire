**При массовом удалении фильтр должен быть ТОЧНЫМ, никогда по подстроке в тексте.**

**Why:** 2026-07-19 владелец велел вычистить коворк от heartbeat-пульсов. Первый проход по
`subject__startswith="HEARTBEAT"` снёс 1203 пульса — правильно. Во втором проходе я добавил
«на всякий случай» `subject__icontains="пульс"` — и удалил **3 ЖИВЫХ сообщения**, где слово
«пульс» просто упоминалось в тексте (включая моё письмо GB с рабочей просьбой). Данные не
восстановить; пришлось пересылать заново.

**How to apply:**
- Удалять только по точному признаку (`startswith` служебного префикса, конкретный тип/флаг),
  никогда по `icontains` слова, которое может встретиться в живом тексте.
- Перед `delete()` — сначала `count()` + вывести примеры того, что попадёт под нож, и сверить
  глазами, что там нет живого.
- Разрушительное — только по явному слову владельца; классификатор, который блокирует массовый
  DELETE, не обходить, а объяснить владельцу и дать решить.
