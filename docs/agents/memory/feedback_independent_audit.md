Аудит AMC P00–P05 (2026-07-05, отдельная Haiku-сессия) нашёл реальные дефекты, которые мои собственные verification pass 1+2 пропустили: Emergency Stop не замораживал серверные дедлайны (только фронтенд), report_count брался из несинхронизированного счётчика вместо реальных строк, очередь модерации фото смотрела не на ту стадию (ingredient_penalty вместо cooking).

**Why:** Самопроверка проверяет то, что построено, по собственной интерпретации критериев. Слепые зоны: (1) "заморозить таймеры" понял как UI, а не серверное время; (2) доверился денормализованному полю; (3) не проследил реальный lifecycle данных (когда именно появляются фото).

**How to apply:** При verification pass 2 явно перечитывать acceptance criteria фазы буквально и спрашивать "что произойдёт с СЕРВЕРНЫМ состоянием, а не с отображением?". Для каждого показываемого числа проверять: это источник истины или кэш/счётчик? Для каждой очереди: на какой стадии реально появляются данные, которые она модерирует? Аудит-коммиты чужой сессии всегда проверять на push (сессия оставила 4 коммита незапушенными). Также: не все находки аудита = дефекты; разделять на (а) реальные дефекты — чинить сразу, (б) законные отсрочки с честным "Unavailable", (в) функции будущих фаз.
