С 2026-07-02 владелец требует фиксировать КАЖДЫЙ шаг работы в трёх местах:
1. **Deployment Journal** — config/release_journal.py (RELEASE_JOURNAL список; страница строится из git-лога, но крупные работы описывать явными записями)
2. **Chef Battle Roadmap** — chef_battle/views.py `_build_battlefield_progress()` (пункты фаз со статусами и датами)
3. **CoWork для greenbear** — САМОЕ ГЛАВНОЕ: приложение /coworking/ (БД на сервере как канал синхронизации агентов, см. [[reference_coworking_app]]); запись через SSH + Python-скрипт (владелец явно авторизовал запись в эти журналы)

**Why:** 2026-07-02 обнаружен откат — работа, сделанная и протестированная владельцем 2026-07-01 (часть хореографии арены, зрительские фичи), пропала из кода. Владелец видел и кликал результат — значит работа существовала и потерялась (вероятно незамерженная ветка/worktree другой сессии). Без журналирования каждого шага такие потери невозможно отследить.

**How to apply:** После каждого значимого шага (не только деплоя): обновить роадмап-пункт в views.py, при деплое — запись в release_journal.py, и запись в CoWork журнал. При старте сессии — проверять CoWork журнал на предмет работы других агентов (Bolt, GreenBear, CrestedTen), прежде чем говорить «не сделано».
