# Рабочая среда пользователя

Пользователь ВСЕГДА работает с виртуальной машины Linux (VirtualBox).

- Промпт выглядит как `deploy@localhost:~$`
- SSH на продакшн-сервер Linode: `ssh deploy@80.85.84.156`
- НЕ предлагать "открой PowerShell" или "Windows Terminal" — пользователь на Linux VM
- Все git-команды, python, manage.py и т.д. выполняются на этой Linux VM

## Правильные пути на сервере

- Проект: `/srv/culineire/current/`
- Деплой: `bash /srv/culineire/current/deploy/update.sh`
- Venv: `/srv/culineire/venv/`
- Статика: `/srv/culineire/shared/staticfiles/`
- Media: `/srv/culineire/shared/media/`
- Secrets: `/srv/culineire/shared/.env`

НЕ использовать `/var/www/` — этого пути не существует на этом сервере.

## Локальная разработка

Файлы проекта шерятся между Windows и Linux VM:
- Windows путь: `E:\CulinEire Project\CulinEire\CulinEire`
- VM путь: `/media/sf_CulinEire`
- Это одни и те же файлы — редактировать можно через любой путь, изменения сразу видны на обеих сторонах

**Локальный dev-сервер НЕ используется** — миграции и зависимости там давно не синхронизированы.
Рабочий флоу: редактировать файлы → `git commit + push` → деплой на прод → проверять на `culineire.ie`.
НЕ предлагать запускать локальный сервер для проверки изменений.
