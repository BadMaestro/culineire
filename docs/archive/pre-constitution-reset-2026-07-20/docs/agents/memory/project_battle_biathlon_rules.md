## Биатлон по ингредиентам

Стадия наступает ПОСЛЕ того как победитель комбата определён. Победитель стреляет по ингредиентам рецепта проигравшего.

### Локи (ставятся при Submission, до начала комбата)
- Проигравший шеф заранее ставит **2 лока** на 2 ингредиента своего рецепта
- Локи зафиксированы до начала боя — нельзя менять после
- Стратегия: лочить самые важные/незаменимые ингредиенты

### Выстрелы победителя
- Победитель делает **3 выстрела** по ингредиентам рецепта проигравшего
- Победитель **не знает** какие ингредиенты залочены (стреляет вслепую)
- Если выстрел попадает в залоченный ингредиент → выстрел **отскакивает**, ингредиент остаётся, попытка **теряется**
- Если оба выстрела попали в залоченные → отскочили оба, третий выстрел идёт по оставшимся

### Возможные итоги (1, 2 или 3 попадания)
- 0 залоченных задеты → **3 ингредиента** убиты/заменены
- 1 залоченный задет → **2 ингредиента** убиты/заменены
- 2 залоченных задето → **1 ингредиент** убит/заменён (третий выстрел попадает)

### Что значит "убит" ингредиент
- Проигравший шеф должен либо **удалить** ингредиент из блюда, либо **заменить** его на другой
- Это ослабляет блюдо проигравшего, но не лишает его шансов — замена может оказаться интересной
- Аудитория голосует за финальное блюдо — слабое блюдо ≠ плохое блюдо

### Принцип механики
- Даёт стимул побеждать в комбате (иначе рецепт пострадает)
- Шефы внимательно относятся к выбору ингредиентов и локов
- Победитель блюда определяется **только аудиторией**, не комбатом

**Why:** Озвучено владельцем 2026-06-11 как финальные правила биатлона.
**How to apply:** При реализации стадии ingredient_selection/biathlon строго следовать этой механике.
